1. Data Catalogue

The Data Catalog is your hub for accessing and uploading spatial data. Here is how to use it:
  • You can click on each dataset to access the metadata and view it in a WebGIS interface.
  • To filter the datasets, click the ‘Filter Button’ and filter by attributes, themes, owner, spatial extent and more.
  • To upload data, click the 'Add Resource' button, and select ‘Upload Dataset’. Drag and drop the desired data into the window and follow the instructions to add a dataset to the platform.
  • Alternatively, you can create a dataset directly on the platform by clicking the ‘Add Resource’ button and selecting ‘Create Dataset’.
  • Finally, you can also clone an existing dataset by clicking the top right corner and selecting ‘Clone’.

2. Maps Section

The Maps Section enables you to access and create both static and interactive maps. Here is what you can do:
  • Access the catalogue of maps, including those created by iMMAP and external organization.
  • To create your own map, click 'Add Resource', or clone an existing map by clicking on the top-right corner of the map feature and selecting ‘Clone’.
  • When you have created an empty map, you can add any dataset from the catalogue using the ‘Add dataset’ button.

3. Dashboards

The Dashboards provide aggregated disaster risk information at both provincial and district levels. Here's what you can do:
  • Access information on people, buildings, and areas exposed to various disaster risks like floods, avalanches, earthquakes, and landslides.
  • View this information through WebGIS viewers, tables, and widgets that quickly display the number of people, buildings, and areas exposed to disaster risk.
  • You can also create your own Dashboard following the same process as for data and maps.

4. Setting Share Permissions

Share permissions control who can view and access your resources (datasets, maps, dashboards, or documents). Here's how you set them:
  • Click the 'Share' button when you have a resource opened.
  • Adjust the permissions for all users on the ‘Anyone’ field. These are the options:
    • None – Nobody can view or download (except for you and iMMAP Admin staff).
    • View – Anyone can view.
    • Download - Anyone can view and download.
  • Alternatively, grant specific permissions to groups or individual users with the +Add User / Group option.
  • By default, permissions are set to 'None', which means that only the data owner (you) and iMMAP admin staff can view or download the resource.
  • All data will be marked by a yellow circle until they have been approved by iMMAP.
